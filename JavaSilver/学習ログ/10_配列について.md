> [!TIP]
> Javaにおける配列の宣言では、[]（ブラケット）を置く場所について、以下の3つのルールが認められている。
> 
> 型の後ろに置く: int[] a; （もっとも一般的なJavaらしい書き方）
> 
> 変数名の後ろに置く: int b[]; （C言語の名残で認められている書き方）
> 
> 両方に分けて置く: int[] e[]; （これでも合計の数が次元数になる）
> 
> 各変数の「次元数」を整理する
> 
> 変数名	書き方	次数	解説
> 
>　int[] a	1次元	基本の書き方。
> 
> int b[]	1次元	変数名の後ろもOK。
> 
> int[][] c	2次元	2次元の基本形。
> 
> int d[][]	2次元	2次元でも変数名の後ろでOK。
> 
> int[] e[]	2次元	型に1つ、名前に1つ。合計で2次元。
> 
> int[][] f[]	3次元	型に2つ、名前に1つ。合計で3次元

> [!WARNING]
> 「コンパイルエラー」になる似たケース1
> ```
> int[5] a; // NG! 宣言時に数字は入れられない
> 配列のサイズを指定できるのは、new int[5] のように インスタンスを作成する時だけ
```


> [!WARNING]
> 「コンパイルエラー」になる似たケース2

> 初期化せずに「使用」する
> ```
int[] a;
System.out.println(a.length); // NG!
ローカル変数の場合、宣言だけでは中身が null すら入っていない「未初期化」状態。宣言自体はエラーになりませんが、その後に使おうとするとコンパイルエラーになる
```

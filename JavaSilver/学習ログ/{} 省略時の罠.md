# {} 省略時の罠

下のコードの出力値は？

```
package sample;

public class Main {

    public static void main(String[] args) {
      if(false)
        System.out.println("A");
      System.out.println("B");
    }
}

答えB
```

## Javaコンパイラには全く違う解釈をされていることに気づかなかった
- if文の後に { }（ブロック）がない
- Javaにおいて、if 文や for 文の後に { } を書かない場合、「直後の1行（1つの文）だけ」 がその制御対象になる、という絶対的なルールがある
### 人間が見ているコード
```
if(false)
  System.out.println("A");
System.out.println("B"); // Aと同じ高さにあるから、仲間かな？と思ってしまう
```
### コンパイラが見ているもの
```
// コンパイラによる脳内補完
if (false) {
    System.out.println("A"); // 直後の1行だけが if の支配下
}

// Bは if とは無関係の、ただの次の行
System.out.println("B");
```


実行の流れ

if(false) の判定：条件は false
System.out.println("A");はスキップ
if 文の処理が終わったので、プログラムは次の行へ
System.out.println("B");
よってBのみ出力
